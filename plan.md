## Market fit

This niche is **evergreen** because proportional relationships, unit rate, and constant of proportionality remain dedicated lesson areas in current 7th-grade math pathways, including current Khan Academy structures and school math unit pages.[^1][^2][^10][^11]
It also has purchase intent on the teacher side, since Teachers Pay Teachers shows active bundles, assessments, comparison resources, and guided-note style products in this exact topic family rather than only broad "middle school math" demand.[^12][^13][^3][^14]

## Implementation guide

The execution order matters more than volume, because current topical-authority advice stresses structured clusters, logical hierarchy, and steady publishing over random content bursts.[^4][^5][^6][^7]
Build in four phases so the site becomes usable, indexable, and monetizable early instead of waiting for a huge "perfect" launch.[^5][^6][^4]

### Phase 1: Foundation, week 1

- Set up Next.js App Router, TypeScript, Tailwind, MDX, and a content collection for articles, tools, and products.
- Create reusable templates for `ToolPage`, `ArticlePage`, `WorksheetPage`, and `HubPage`.
- Create one design system only: teal/ink/off-white, graph-paper accents, simple math diagrams.
- Add global SEO defaults: canonical, Open Graph, Twitter, robots, sitemap, breadcrumbs, and organization schema.


### Phase 2: MVP authority, weeks 2–4

- Publish homepage.
- Publish the four pillar pages.
- Publish the first six tools.
- Publish the first eight articles.
- Add Search Console, GA4, Bing Webmaster Tools, and Microsoft Clarity.
- Submit XML sitemap immediately.


### Phase 3: Programmatic expansion, weeks 5–8

- Add tool-content pairs, for example:
- calculator + explainer
- checker + mistake article
- worksheet + answer key page
- Build a JSON source file for reusable examples so each tool can output:
- answer
- steps
- mistake notes
- FAQ
- related links
- Add schema at page level:
- Tool pages: `WebApplication`
- Articles: `Article`
- FAQ blocks: `FAQPage`
- Navigation: `BreadcrumbList`


### Phase 4: Store and trust, weeks 9–12

- Launch `/worksheets` and `/resources/teachers`.
- Add teacher-facing product pages with previews, standards alignment, answer keys, and classroom use notes.
- Add author pages, editorial standards, "how examples are checked," and "who this site helps" trust pages to strengthen expertise signals.[^7]

## Monetization

Display ads alone can work, but the stronger play is stacked monetization because current publisher benchmarks say general content sites often land around $5–$15 RPM, AdSense may run at roughly 30–50% of premium-network rates, and premium networks can materially lift earnings once traffic grows.[^15]
The highest-margin revenue layers are still affiliate, digital products, and email upsells, which fits this niche well because you can serve students, parents, tutors, and teachers from the same topic base.[^8][^9][^15]

### Monetization stack

| Stream | What to sell | Page types | Notes |
| :-- | :-- | :-- | :-- |
| Display ads | AdSense first, premium network later | Articles, hubs, worksheets | Keep tool UI clean above the fold; ads below calculator result |
| Tutoring affiliate | Wyzant partner links | Homework-help pages, "need a tutor?" blocks | Wyzant pays $8 per qualified student lead referred from your site. [^9] |

For Mathway, Photomath, and similar brands, validate live partner terms before building core funnel pages around them, but keep "homework help," "show me the steps," and "find a tutor" CTAs ready so any approved offer can plug into the same template.
